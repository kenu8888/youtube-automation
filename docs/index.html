撮影機材

カメラとマイクが直接つなげないので、別々に録って後からソフトで合わせています。

機材一覧
機材 モデル 備考
カメラ Sony a7C 4K XAVC S / H.264 / 23.976fps
マイク FIFINE K688CTW ダイナミックマイク、USB-CでPC録音
録音 iPad ボイスメモ m4a / AAC 44.1kHz（暫定、K688に切替予定）
なぜ別撮りか

K688CTWの出力はXLRまたはUSB-Cのみ。Sony a7Cのマイク入力は3.5mmミニジャック。物理的に直接接続できないため、映像と音声を別々に収録し、パイプラインのステップ1（音声同期）でソフトウェア的に結合しています。

拍手を1回入れてピーク波形を作り、相互相関でオフセットを自動検出する方式を採用しています。

ステップ 1 — 音声同期

カメラの音声と外部マイクの音声を自動でぴったり合わせる。

ざっくり言うと
カメラと外部マイクを別々に録音しているので、映像と音声がズレています。撮影時に「パン！」と手を叩いておくと、その音の波形を手がかりにして自動でタイミングを合わせてくれます。

やること： python3 audio_sync.py 映像.MP4 音声.m4a を実行するだけ。
仕組み（技術詳細）
1
音声抽出
映像ファイルと音声ファイルからそれぞれモノラル16kHz WAVを抽出。FFmpegで変換。
2
オフセット検出
SciPyのcorrelate()で相互相関を計算し、ピーク位置から時間オフセットを算出。拍手の波形が合致するポイントを自動検出する。
3
映像+音声マージ
検出したオフセットに基づき、外部音声をトリムまたはパディングしてから映像と結合。映像は再エンコードしない（-c:v copy）。
技術詳細
項目 値
同期検出サンプルレート 16,000 Hz（高速化のため低解像度で処理）
同期方式 相互相関（scipy.signal.correlate, mode="full"）
同期精度 1/16000秒 = 0.0625ms
音声コーデック（出力） AAC 192kbps
依存ライブラリ numpy, scipy, ffmpeg（CLI）
使い方
# 依存インストール pip install numpy scipy # 実行（音声同期 + 音声最適化） python3 audio_sync.py 映像.MP4 音声.m4a # 音声最適化をスキップ（同期のみ） python3 audio_sync.py 映像.MP4 音声.m4a --skip-normalize # 出力ファイル名を指定 python3 audio_sync.py 映像.MP4 音声.m4a -o output.mp4

ステップ 2 — 音声最適化

音をきれいに、ちょうどいい音量に自動調整する。

ざっくり言うと
撮影した音声には3つの問題があります：
1. エアコンや換気扇の「サー」というノイズが入っている
2. 拍手の音が大きすぎて、喋りの音が小さく聞こえる
3. 他のYouTube動画と比べて全体的に音量が小さい

この3つを自動で直します。音声同期のスクリプトに組み込み済みなので、追加の操作は不要です。
課題

撮影環境の換気扇によるホワイトノイズが混入し、また音声同期に使う拍手のピークが大きいため、喋り部分の音量が相対的に小さくなる問題があった。他のYouTube動画と比較して約70%程度の音量しかなかった。

処理チェーン
1
ノイズ除去 afftdn
FFTベースのノイズ除去。換気扇等の定常ノイズ（ホワイトノイズ）を検出して除去する。nr=10で控えめに効かせ、声への影響を最小限に。
FFmpeg
2
コンプレッション acompressor
拍手等のピークを抑え、喋りの音量を相対的に持ち上げる。threshold=-25dB、ratio=4:1、makeup=8dB。
FFmpeg
3
ラウドネス正規化 loudnorm
YouTube基準の-14 LUFSに正規化。トゥルーピーク上限-1.5dBでクリッピングを防止。
FFmpeg
処理結果
指標 処理前 処理後
平均ラウドネス -35.51 LUFS -14.93 LUFS
トゥルーピーク -1.50 dB -0.85 dB
ラウドネスレンジ 9.60 LU 7.60 LU
パラメータ詳細
フィルタ パラメータ 説明
afftdn nf=-25:nr=10:nt=w ノイズフロア-25dB、ノイズリダクション10dB、ホワイトノイズタイプ
acompressor threshold=-25dB:ratio=4:attack=5:release=50:makeup=8 -25dB超の音を4:1で圧縮、アタック5ms、リリース50ms、メイクアップ8dB
loudnorm I=-14:TP=-1.5:LRA=11 目標-14 LUFS、ピーク上限-1.5dB、ラウドネスレンジ上限11LU
FFmpegコマンド（単体実行する場合）
ffmpeg -i input.mp4 \ -af "afftdn=nf=-25:nr=10:nt=w,acompressor=threshold=-25dB:ratio=4:attack=5:release=50:makeup=8,loudnorm=I=-14:TP=-1.5:LRA=11" \ -c:v copy \ output.mp4
音声同期スクリプト（audio_sync.py）に統合済み。--skip-normalize フラグで音声最適化をスキップ可能。
